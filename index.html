2. 한상훈 의사결정 3대 어젠다

어젠다핵심 관점의사결정 반영
한상훈의 역량통계·데이터 구조화·SaaS CSM·SLA/VOC·리포팅에 강함. 직접 모객·환자 클로징은 검증 필요.기본 포지션은 데이터 아키텍터 / CRM Data Ops로 정의
최준혁 대표의 요구일본 모객→상담→병원상담→예약→방한·시술→결제→사후관리까지 End-to-End 운영 오너 요구.요구 범위는 유지하되, 3개월 평가에서는 직접 매출 KPI보다 운영체계·추적구조를 중심으로 조정
용성남 판단데이터 아키텍터로 보되 운영씬의 역할을 최대한 챌린지하여 Fit과 결과를 검증.3개월 한시 채용 + 30/60/90일 Gate Review + 정규 전환 조건 명확화

4. 신규 후보자 페이지 추가 방법

  1. hr/interviewees/template.html 파일을 복사합니다.
  2. 파일명을 후보자 영문명 또는 한글명 영문표기로 저장합니다. 예: kimyounghee.html
  3. 템플릿 안의 후보자명, 포지션, 평가 점수, 의사결정 내용을 수정합니다.
  4. hr/index.html의 인터뷰이 평가자료 섹션에 후보자 카드를 추가합니다.
  5. GitHub에 업로드 후 Pages URL에서 정상 노출되는지 확인합니다.
주의: GitHub Pages의 정적 비밀번호 게이트는 강력한 보안이 아닙니다. 민감 자료는 private repository, 별도 인증 서버, Notion 권한관리, Google Drive 권한공유 등을 사용하세요.
